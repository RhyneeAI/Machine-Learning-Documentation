01 Label Encoding

Ganti kategori dengan angka integer berurutan.

Merah → 0
Hijau → 1
Biru → 2
✓ Kolom target / data ordinal
✗ Nominal → model salah anggap ada urutan
02 One-Hot Encoding

Setiap kategori jadi kolom baru berisi 0 atau 1.

_Merah _Hijau _Biru
Merah → 1   0   0
Hijau → 0   1   0
Biru → 0   0   1
✓ Nominal, jumlah kategori sedikit
✗ Banyak kategori → dimensi meledak
03 Ordinal Encoding

Seperti label encoding, tapi mapping mengikuti urutan yang bermakna.

SMA → 1
D3 → 2
S1 → 3
S2 → 4
✓ Data punya hierarki / tingkatan jelas
✗ Jangan pakai jika urutan tidak natural
04 Binary Encoding

Kategori → angka → representasi biner. Hemat kolom vs One-Hot.

Merah → 1 → 001
Hijau → 2 → 010
Biru → 3 → 011
Kuning → 4 → 100
✓ Kategori nominal banyak nilai unik
✗ Lebih kompleks, kurang intuitif
Panduan Cepat — Kapan Pakai Yang Mana?
Binary (2 nilai) → Label Nominal, sedikit nilai → One-Hot Ada urutan jelas → Ordinal Nominal, banyak nilai → Binary
